AWS 支持多种Cloud Volumes ONTAP配置。



== 支持的节点数

Cloud Volumes ONTAP在 AWS 中可作为单节点系统使用，也可作为高可用性 (HA) 节点对使用，以实现容错和无中断操作。

不支持将单节点系统升级到 HA 对。如果您想在单节点系统和 HA 对之间切换，则需要部署新系统并将数据从现有系统复制到新系统。



== 支持的存储

Cloud Volumes ONTAP支持多种类型的 EBS 磁盘和 S3 对象存储，用于数据分层。最大存储容量由您选择的许可证决定。



=== 通过许可证支持存储

每个许可证支持不同的最大系统容量。最大系统容量包括基于磁盘的存储以及用于数据分层的对象存储。  NetApp不支持超出此限制。

== 支持的 EC2 计算

每个Cloud Volumes ONTAP许可证支持不同的 EC2 实例类型。为了您的方便，下表显示了每个受支持的实例类型的 vCPU、RAM 和带宽。 https://aws.amazon.com/ec2/instance-types/["您应该参考 AWS 以获取有关 EC2 实例类型的最新完整详细信息"^] 。

Cloud Volumes ONTAP可以在预留或按需 EC2 实例上运行。不支持使用其他实例类型的解决方案。

. 一些实例类型包括本地 NVMe 存储， Cloud Volumes ONTAP将其用作_Flash Cache_。 Flash Cache 通过实时智能缓存最近读取的用户数据和NetApp元数据来加快数据访问速度。它对于随机读取密集型工作负载（包括数据库、电子邮件和文件服务）非常有效。必须在所有卷上禁用压缩才能利用 Flash Cache 性能改进。 https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-flash-cache.html["了解有关闪存的更多信息"^] 。
. 使用 HA 对时， Cloud Volumes ONTAP支持大多数实例类型的高写入速度。使用单节点系统时，所有实例类型都支持高写入速度。 https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-write-speed.html["了解有关选择写入速度的更多信息"^] 。
. r5.12xlarge 实例类型具有已知的可支持性限制。如果节点由于崩溃而意外重启，系统可能无法收集用于排除故障和找出问题根源的核心文件。如果发生这种情况，客户将接受风险和有限的支持条款并承担所有支持责任。此限制影响新部署的 HA 对和从 9.8 升级的 HA 对。该限制不会影响新部署的单节点系统。
. 虽然这些 EC2 实例类型支持超过 64 个 vCPU，Cloud Volumes ONTAP仅支持最多 64 个 vCPU。
. 当您选择 EC2 实例类型时，您可以指定它是共享实例还是专用实例。
. 以下 EC2 实例类型系列（大小从 xlarge 到 4xlarge）支持 AWS 本地区域：M5、C5、C5d、R5 和 R5d。link:https://aws.amazon.com/about-aws/global-infrastructure/localzones/features/?nc=sn&loc=2["您应该参考 AWS 以获取有关本地区域中支持的 EC2 实例类型的最新完整详细信息"^] 。
+
AWS 本地区域中的这些实例类型不支持高写入速度。





=== 不再支持 c4、m4 和 r4 实例

Cloud Volumes ONTAP不再支持 AWS 中的 c4、m4 和 r4 EC2 实例类型。如果您的系统在 c4、m4 或 r4 实例上运行，请更改为 c5、m5 或 r5 实例。除非您更改实例类型，否则您无法升级到此版本。

link:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-change-ec2-instance.html["了解如何更改Cloud Volumes ONTAP的 EC2 实例类型"^] 。

有关详细信息，请参阅：

* https://kb.netapp.com/Cloud/Cloud_Volumes_ONTAP/Unable_to_change_the_instance_type_from_r4_to_r5_with_disk_count_error["知识库文章：无法将实例类型从 r4 更改为 r5，并且出现磁盘数量错误"^]
